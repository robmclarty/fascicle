# Roles and permissions

Skylark has three workspace roles: admin, editor, and viewer. Roles apply to
the whole workspace; there are no per-project roles.

## What each role can do

Admins manage the workspace itself: settings, members, billing, and deletion.
Editors create and change content: projects, boards, and tasks. Viewers have
read-only access to everything except billing.

## Deleting a project

Only workspace admins can delete a project. Editors can archive a project,
which hides it from the sidebar without removing data. Deleting a project
moves it to a 30-day recovery window before it is removed permanently.

## Changing someone's role

Admins change roles from the Members page. A workspace must always have at
least one admin; the last admin cannot be demoted or removed.
