# AI Engine Build — Handoff

## Phase 1: Engine Package Foundation

### What was built

- **`packages/engine/` workspace package** — ESM, TypeScript strict, NodeNext, ES2024. `package.json` declares `ai`@^6 and `zod`@^4 as the only production dependencies; the six provider SDKs (`@ai-sdk/anthropic`, `@ai-sdk/openai`, `@ai-sdk/google`, `ai-sdk-ollama`, `@ai-sdk/openai-compatible`, `@openrouter/ai-sdk-provider`) are declared as optional peers via `peerDependenciesMeta`. The workspace root already globs `packages/*` so no additional registration was needed.
- **`types.ts`** — every public type from spec §5: `GenerateOptions`, `GenerateResult`, `Message`, `UserContentPart`, `AssistantContentPart`, `Tool`, `ToolExecContext`, `StreamChunk`, `ToolCallRecord`, `StepRecord`, `UsageTotals`, `CostBreakdown`, `FinishReason`, `EffortLevel`, `RetryPolicy`, `AliasTarget`, `AliasTable`, `Pricing`, `PricingTable`, `EngineConfig`, `ProviderConfigMap`, `ProviderInit`, `Engine`, `ToolApprovalHandler`, `ToolApprovalRequest`. Types PascalCase, fields snake_case, `@robmclarty/core` imported with `import type` only.
- **`errors.ts`** — the 11 typed error classes, each extending `Error` with a `readonly kind` discriminant and the documented metadata fields. Exception to `no-class` rule via the extended ignore list.
- **`aliases.ts`** — frozen `DEFAULT_ALIASES` matching spec §5.7 entry-for-entry plus `resolve_model`. The colon-prefix path splits on the **first** colon only, so `openrouter:anthropic/claude-sonnet-4.5` resolves to `{ provider: 'openrouter', model_id: 'anthropic/claude-sonnet-4.5' }`.
- **`pricing.ts`** — frozen `DEFAULT_PRICING` (Anthropic, OpenAI, Google; none for Ollama, LM Studio, OpenRouter), frozen `FREE_PROVIDERS` set (ollama, lmstudio), and `compute_cost` implementing the §5.10 formula (fresh-input subtracts cache reads and writes; reasoning rolls into `output_usd` when no separate rate; 6-decimal rounding; zero-usage components omitted, not zeroed; missing pricing for a free provider returns all-zero; missing pricing for a paid provider returns `undefined`).
- **`retry.ts`** — `DEFAULT_RETRY` + `parse_retry_after` (numeric seconds and HTTP-date) + `retry_with_policy` with jittered exponential backoff, abort-interruptible `wait_ms`, and `Retry-After` honoring.
- **`streaming.ts`** — `normalize_chunk(event, step_index)` mapping AI SDK v6 events to `StreamChunk`, plus `create_chunk_dispatcher(on_chunk)` catching sync throws and rejected promises, wrapping in `on_chunk_error`, and signaling `aborted()` for the orchestrator.
- **`trajectory.ts`** — no-op when `trajectory` is `undefined`; otherwise opens `engine.generate` and `engine.generate.step` spans and records `request_sent`, `response_received`, `tool_call`, `cost`, `pricing_missing` (via `create_pricing_missing_dedup`), `effort_ignored`, `tool_approval_requested`, `tool_approval_granted`, `tool_approval_denied`.
- **`usage.ts` / `schema.ts`** — `sum_usage(steps)` treats absent as 0 for arithmetic but leaves fields `undefined` on the aggregate when no step reported them. `parse_with_schema` returns a tagged union; `build_repair_message` produces the canonical §6.5 repair user message; `throw_schema_validation` throws `schema_validation_error` carrying the zod error and raw text.
- **`providers/`** — shared `types.ts` (`ProviderAdapter`, `load_optional_peer`, `default_normalize_usage`), `registry.ts` (builtin map + `get_provider_factory` throwing `provider_not_configured_error` on miss), and adapters for Anthropic / OpenAI / Google / Ollama / LM Studio / OpenRouter. Each adapter validates its credentials synchronously (throwing `engine_config_error`), exposes `build_model(model_id) → Promise<unknown>` that dynamically imports its peer SDK, and translates `EffortLevel` per §6.3.
- **Architectural rules** — `rules/no-class.yml` now ignores `packages/engine/src/errors.ts`; `no-default-export.yml` already covered the engine via the `packages/*/src/**/*.ts` glob; `no-process-env-in-core.yml` and `snake-case-exports.yml` scopes extended to engine; new `rules/no-core-value-import-in-engine.yml` (forbids value imports from `@robmclarty/core`, permits `import type`); new `rules/no-provider-sdk-outside-providers.yml` (forbids `@ai-sdk/*`, `ai-sdk-ollama`, `@openrouter/ai-sdk-provider` imports outside `packages/engine/src/providers/**`).
- **`scripts/check-deps.mjs`** — extended to assert engine's production deps are exactly `ai` + `zod` and that all six provider SDKs are declared as optional peers.
- **Colocated unit tests** — every module has a sibling `*.test.ts`; see `packages/engine/src/**/*.test.ts`. Includes `providers/types.test.ts` with the definitive `load_optional_peer` missing-peer assertion (see Decisions below).

### Decisions

- **Adapter `build_model` returns `Promise<unknown>`, not `LanguageModel`**: tension between criterion 12 ("adapter exposes non-streaming generation, streaming generation, effort translation") and invariant 13 ("only generate.ts / tool_loop.ts / index.ts may call generateText/streamText from `ai`"). Resolved by making adapters thin: they build and return the provider-native `LanguageModel` via dynamic import, and phase 2's orchestrator calls `generateText` / `streamText` on the returned model. This keeps the invariant-13 boundary clean while still giving adapters sole ownership of effort translation and usage normalization.
- **Synchronous factory + async `build_model`**: factories are synchronous so `create_engine(config)` can validate all provider init values at construction time (throwing `engine_config_error` on missing credentials) without touching the network or the filesystem. SDK loading is deferred to the first `generate(...)` call so missing-peer failures are surfaced with the request, not with engine construction.
- **Retry-After outranks `max_delay_ms`**: when a 429 carries a numeric or HTTP-date `Retry-After`, the backoff delay is `max(computed_backoff, retry_after_ms)` without capping by `max_delay_ms`. The server's explicit retry instruction outranks the local jitter cap. The backoff cap still applies for retries without a server-supplied hint.
- **Missing-peer test lives in `providers/types.test.ts`, not per-adapter**: the six provider SDKs are installed as optional peers in `packages/engine/node_modules/` during workspace setup, so a per-adapter "peer missing" test against `adapter.build_model(...)` cannot run in this environment. The definitive assertion covers the same code path (`load_optional_peer` wrapping an `ERR_MODULE_NOT_FOUND` into a clear error naming the missing package) via a bogus specifier in `providers/types.test.ts`. Per-adapter tests instead assert successful `build_model` resolution as a sanity check.
- **`load_optional_peer` uses an inline `eslint-disable` for `no-unsafe-type-assertion`**: the function is a generic boundary cast (`unknown` → `T`) and that boundary can't be made safer without a runtime schema for every peer SDK. The disable is scoped to one line; `Error` is constructed with `{ cause: err }` so the original error is preserved.
- **`classify_retryable` uses `Reflect.get` instead of a record cast**: oxlint's `no-unsafe-type-assertion` flagged the narrow assertion from `unknown` to `Record<string, unknown>`. `Reflect.get(obj, key)` already returns `unknown`, so the local helpers `read_string` / `read_number` operate on a typed-narrow `object` input and avoid the cast entirely.

### Deviations

- None from the phase spec. Error class names are snake_case (`aborted_error`, `rate_limit_error`, …) to match the existing convention in `packages/core/src/errors.ts` (the `no-class` rule's existing exception). The `no-class` rule's ignore list now permits both files by exception.

### Notes for next phase

- **Phase 2 owns orchestration**: `create_engine.ts`, `generate.ts`, `tool_loop.ts`, and the `index.ts` re-export surface. Phase 2 is the only place permitted to call `generateText` / `streamText` from `ai` (invariant 13).
- **Effort-ignored signal**: when `ProviderAdapter.translate_effort(effort).effort_ignored` is `true`, the orchestrator must emit `record_effort_ignored(trajectory, model_id)`. Ollama and LM Studio set this to `true` for any non-`none` effort; OpenRouter and all reasoning-capable providers forward effort to the SDK.
- **Retry boundary for streaming**: `retry_with_policy` is designed to wrap a single provider call. Phase 2 must NOT wrap streaming responses past the first chunk — once bytes have been delivered to `on_chunk`, a retry would re-emit them. The generate orchestrator should set a "streamed-once" boundary before entering the stream loop.
- **Schema repair loop**: `parse_with_schema` + `build_repair_message` + `throw_schema_validation` are primitives only. The repair loop itself (counting toward `max_steps`, re-dispatching through the provider with the repair message appended) lives in phase 2.
- **Pricing-missing dedup**: `create_pricing_missing_dedup(trajectory)` returns an object with `emit(provider, model_id)` that fires at most once per `{provider, model_id}` per call. Phase 2 should instantiate it once per `generate(...)` invocation so the warning is emitted once regardless of step count.
- **`engine_config_error` vs `provider_not_configured_error`**: the former is thrown by adapter factories when required credentials are missing (e.g., empty `api_key`). The latter is thrown by `get_provider_factory(name)` when the provider name isn't in `BUILTIN_PROVIDERS`. Phase 2's `create_engine(config)` should let both propagate from `get_provider_factory(name)(init)`.
- **`pnpm check` exits 0** with phase 1's files in place. All 8 checks green: types, lint, struct, deps, dead, test, docs, spell. Coverage numbers are in `.check/test.json`.

## Phase 2: Generate Orchestration, Engine Factory, and Full Validation

### What was built

- **`create_engine.ts`** — synchronous factory that validates credentials at construction time (provider adapters' factories throw `engine_config_error` for missing/empty api_key), builds the adapter map, merges user-supplied aliases over `DEFAULT_ALIASES`, merges user-supplied pricing over `DEFAULT_PRICING`, and returns a frozen `Engine` with a bound `generate`. Unconfigured providers surface as `provider_not_configured_error` at call time, not construction.
- **`generate.ts`** — single public entry point for model calls. Resolves the model alias, selects the adapter, checks capabilities (`schema`, `tools`, `streaming`) and throws `provider_capability_error` on mismatch, translates effort (`record_effort_ignored` fires when the adapter ignores it), builds initial `ModelMessage[]` (system/user/assistant/tool content shapes), prepends the schema-response prefix when `opts.schema` is present, and drives `run_tool_loop` via an `InvokeOnce` seam. The seam owns the `ai` SDK boundary: `collect_stream` (streamText) and `collect_non_stream` (generateText) are the only call sites. Retry wraps each provider call via `retry_with_policy`; abort links the user's `AbortSignal` into an internal controller so cleanup can cancel in-flight streams on user abort. Cost aggregation (`aggregate_cost`) adds per-step breakdowns into a single `GenerateResult.cost`, rounding to 6 decimals and omitting zero-usage components. Free providers (`ollama`, `lmstudio`) resolve missing pricing to all-zero `is_estimate: true`.
- **`tool_loop.ts`** — sequential per-turn tool execution with zod validation (invalid input is fed back with `error: true` and consumes a step; `execute` is not called), `needs_approval` gating (absent handler fails closed with `tool_approval_denied_error`; abort during the await rejects with `aborted_error`), `tool_error_policy: 'feed_back' | 'throw'`, `max_steps` cap resolving with `finish_reason: 'max_steps'` (attempted-but-unexecuted tool calls from the final turn land in `tool_calls` with a canonical error marker), per-step cost computation + trajectory recording, `pricing_missing` dedup via phase-1's factory, and stream-parity guarantee: a streamed call returns a `GenerateResult` deep-equal to the non-streamed call for the same provider response (modulo `duration_ms` / `started_at` on tool_calls).
- **`index.ts`** — full public surface: `create_engine` plus every type from spec §5 and every typed error from spec §9. No provider adapter or internal orchestration helper is re-exported.
- **`@robmclarty/agent-kit` umbrella** — `packages/agent-kit/src/index.ts` re-exports `create_engine` alongside `@robmclarty/core`, plus both type namespaces (`EngineConfig`, `GenerateOptions`, `GenerateResult`, `Engine`, `Tool`, …) and every typed error (`aborted_error`, `provider_error`, `rate_limit_error`, …). `index.test.ts` asserts presence of the engine surface.
- **Test coverage** — spec §10 success criteria and spec §9 failure modes are covered across:
  - `generate.test.ts`, `tool_loop.test.ts`, `create_engine.test.ts`, `version.test.ts`, `frozen.test.ts` (co-located unit tests)
  - `src/spec_coverage.test.ts` (24 tests for remaining success criteria — C7, C8, C9, C12, C14, C17, C18, C20, C23, C27, C28, C30, C32 — and failure modes F1, F2, F6, F7, F12, F13, F14, F19, F22)
  - `test/integration/streaming_parity.test.ts` (streamed vs non-streamed GenerateResult deep-equal)
  - `test/integration/with_composition.test.ts` (cross-layer: chunks flow through `ctx.emit` into `run.stream`, `timeout(slow_step, 50)` propagates abort into engine, LIFO cleanup fires on engine rejection, cost events reach the userland trajectory observer)
  - `test/cleanup/sigint.test.ts` + harness (`child-harness.ts`, `ai-stub.mjs`, `sdk-stub.mjs`, `ts-resolver.mjs`, `register-ts-resolver.mjs`) — SIGINT during a stubbed hanging `generate` aborts the runner, propagates through the engine's internal controller, rejects with `engine_aborted_error`, fires LIFO cleanup handlers, and exits non-zero. The stub's `hang_until_abort` uses a `setInterval` keepalive because Node 24 exits code 13 ("unsettled top-level await") when only an event listener is active.
- **`test/fixtures/mock_ai.ts`** — shared `vi.mock` factories for the `ai` module and `./providers/registry.js`. Queues `generateText` responses and `streamText` scripts (array of parts plus optional per-part delay for abort-timing tests), captures call counts and last `AbortSignal` for retry/abort assertions, and supports per-adapter `capability_overrides` so tests can simulate non-streaming adapters (F6) without touching the real factory registry.

### Decisions

- **Schema repair is wired inline in `generate.ts`**: `schema_repair_attempts` counts from 0 and is decremented on each `schema_validation_error`; the repair user message (`build_repair_message`) is appended to `messages`, and another `generate` turn runs. Exhausted attempts surface the last `schema_validation_error`. The schema-response prefix is injected into the system message (or prepended as a fresh system message if none exists) so the model sees the JSON-only constraint on every attempt.
- **Cost aggregation treats missing-per-step as "unpriced"**: `aggregate_cost(steps, provider)` sums only the steps that reported cost; it returns `undefined` when **none** reported cost (for paid providers) or an all-zero `is_estimate: true` breakdown for free providers. It does not coerce missing per-step costs to zero — the `pricing_missing` event is the signal for that, emitted at most once per call per `{provider, model_id}`.
- **Streaming-abort seam**: the internal `AbortController` fires on either (a) the user's `AbortSignal` transitioning to `aborted`, or (b) an `on_chunk` handler throwing. In case (b), the in-flight stream is cancelled immediately and the thrown error is re-raised as `on_chunk_error` (not wrapped further by retry). Retry checks `err instanceof on_chunk_error` and `err instanceof aborted_error` first and rethrows without retrying.
- **`base_params` typed as `Parameters<typeof streamText>[0] & Parameters<typeof generateText>[0]`**: the SDK's param types differ in options, so using the intersection lets the same object flow to either call site without an unsafe `Record<string, unknown>` → specific-type cast. `providerOptions` still requires one `eslint-disable-next-line typescript-eslint/no-unsafe-type-assertion` at the assignment site because `Record<string, Record<string, unknown>>` is not structurally assignable to `Record<string, JSONObject>` (the SDK's `JSONObject` index signature disallows `unknown`). The disable is scoped to one line and documented.
- **Free-provider zero-cost is only emitted when at least one step ran**: `aggregate_cost` returns `undefined` for a free provider with zero steps (defensive; `generate` always produces at least one step in practice), and an all-zero `is_estimate: true` breakdown otherwise. This keeps `GenerateResult.cost` consistent with the per-step semantics — free providers get zeros, paid providers with missing pricing get `undefined` + a `pricing_missing` event.
- **Cross-layer integration test uses `timeout(slow_step, 50)` for abort propagation**: `run(step)` does not expose its controller externally, so a test that fires SIGINT would need a child process. The `timeout` composer fires an internal abort deterministically after a delay, and the stream script is given a 500 ms per-part delay — abort wins, `aborted_error` surfaces from within `engine.generate`, and the outer `run(bounded, ...)` rejects.

### Deviations

- **Fallback for `unknown` in `safe_json_stringify`**: the helper uses `JSON.stringify(value) ?? String(value)` rather than a ternary. Behavior is unchanged (string-or-undefined coercion) — this was a mechanical adjustment for `prefer-nullish-coalescing`.
- **Tool input/approval no longer cast to `never`**: previously `tool.execute(validation.value as never, tool_ctx)` and `tool.needs_approval(input as never)` were used to silence generic-parameter contravariance errors from `Tool<i, o>`. Now the function references are captured into a local before invocation (`const execute = tool.execute; execute(validation.value, tool_ctx)`), which narrows the apparent parameter to `unknown` without a narrowing cast. Runtime behavior is identical.
- **`User` message content supports `UserContentPart[]` explicitly**: previously `m.content` was cast `as string` even when the `Message` type permits `string | UserContentPart[]`. `to_sdk_messages` now maps the array branch into SDK-shaped `TextPart`/`ImagePart` entries; `media_type` is omitted when absent (required under `exactOptionalPropertyTypes: true`).

### Notes for next phase

- **Phase 2 closes the spec**: every spec §10 success criterion and spec §9 failure mode is exercised through `generate` end-to-end. Any follow-on work is ergonomics, not spec coverage.
- **`create_engine` / `generate` are the ONLY public entry points** for model calls. The `Engine` type is intentionally narrow: `{ generate, describe_aliases(), describe_pricing() }`. Userland tools compose around it via `@robmclarty/core.step` and `run`.
- **Trajectory is a pure observation seam**: no generate code path depends on the presence of a `TrajectoryLogger`. Pass `trajectory: ctx.trajectory` from a `core.step` and engine spans nest automatically under the step span in the run's unified trajectory tree.
- **Adapter capability probing**: tests use `mock_state.capability_overrides` to simulate providers that don't support streaming/tools/schema. Real adapters declare capabilities statically; changing them requires an adapter-source change.
- **Streaming-parity invariant is load-bearing**: compositions that read `GenerateResult` must be able to switch between streaming (via `on_chunk`) and non-streaming without behavior change. Phase 2's `spec_coverage.test.ts` and `streaming_parity.test.ts` regression-test this explicitly.
- **SIGINT harness uses a custom ESM resolver** (`test/cleanup/ts-resolver.mjs`) to remap `ai` → `ai-stub.mjs` and `@ai-sdk/anthropic` → `sdk-stub.mjs` in the child process, plus `.js → .ts` sibling resolution so the harness can import engine/core source directly. Future harnesses should follow the same pattern rather than adding new build steps.

## Phase 2 (retry 1): F10 coverage gap

### What was built
- Added `F10 token limit exceeded mid-stream returns finish_reason length with partial content` to `packages/engine/src/spec_coverage.test.ts` in the `spec §9: failure modes (remaining)` block (positioned between F7 and F12). Asserts both branches: (a) without schema, `result.finish_reason === 'length'` and `result.content` retains the partial assistant text verbatim; (b) with `schema` set, `result.finish_reason === 'length'` and `result.content` is the raw unparsed string — `generate` does NOT attempt schema repair when `finish_reason !== 'stop'` (generate.ts:615), so no `schema_validation_error` is thrown and the truncated JSON fragment passes through as-is.

### Decisions
- **Schema branch returns raw string on `finish_reason: 'length'`**: the phase-2 implementation short-circuits schema parsing the moment the turn doesn't end with `'stop'` (generate.ts:614-615). The F10 test now pins this contract: callers who hit the token cap receive the partial text under `content`, and it is their responsibility to treat it as truncated. This matches spec §9 F10's documented behavior (partial content preserved, not an exception).

### Deviations
- None.
