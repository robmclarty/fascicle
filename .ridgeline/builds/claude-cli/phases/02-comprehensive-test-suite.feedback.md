# Reviewer Feedback: Phase 02-comprehensive-test-suite

## Failed Criteria

### Criterion 4
**Status:** FAIL
**Evidence:** auth.test.ts covers build_env under all three modes, the api_key validator, CLI_AUTH_ERROR_PATTERNS matching, and frozen constants. However the env-scrub test is a direct build_env unit test (auth.test.ts:58–74) rather than the required environment-echoing fixture invocation: 'verified via an environment-echoing fixture' is explicit in this criterion and is not satisfied by unit-level coverage.
**Required state:** A named test exists that creates the claude_cli adapter with auth_mode='oauth' and provider config.api_key set, calls generate with call_opts.env.ANTHROPIC_API_KEY supplied, and inspects the mock's recorded env (or echoed stdout) to assert ANTHROPIC_API_KEY is absent.

### Criterion 7
**Status:** FAIL
**Evidence:** integration.test.ts #1 asserts cross-layer cost with source='provider_reported' (lines 103–111) and the SIGINT harness verifies LIFO cleanup markers plus aborted_error propagation. But the test does NOT verify the trajectory contains an 'engine.generate' parent span ending with {error} or at least one 'engine.generate.step' child span — only cost events are filtered (line 103). The capture logger's start_span records no names; spans are issued but not asserted. This sub-requirement of criterion 7 is missing.
**Required state:** The integration test captures span names (e.g. using the fixtures/mock_helpers.create_captured_trajectory which records spans[].name and ended) and asserts, after an abort, that there is a span with name='engine.generate' whose ended metadata includes an error, and at least one span with name='engine.generate.step'.

### Criterion 10
**Status:** FAIL
**Evidence:** F18 (spawn.test.ts wait_close ENOENT), F19 (auth.test.ts CLI_AUTH_ERROR_PATTERNS loop), F20 (auth.test.ts empty api_key), F21/F22 (cancellation.test.ts timeouts), F24 (stream_parse malformed + cancellation no_result_event), F28 (abort mid-stream, mislabeled F27 in the cancellation.test.ts tag), F29 (dispose) — present. MISSING: F23 'retry per retry_policy' has no named test; F25 'allowlist_only silent drop with trajectory record' has no test (adapter emits kind: 'cli_tool_bridge_allowlist_only' at index.ts:318 but it is never asserted); F26 'forbid pre-spawn throw' has no test (adapter throws at index.ts:300–307 but never exercised); F27 'multi-turn Message[] without session_id' has no test (the F27 tag in cancellation.test.ts:58 actually covers F28 abort-during-subprocess-run); F30 'sandbox binary missing' has no test. README.md:84–88 misattributes F25/F26/F27/F28/F30 to unrelated tests.
**Required state:** A named test (it/test) exists that configures retry_policy, drives the mock to exit non-zero on first attempt and succeed on retry (or exhausts retries and surfaces claude_cli_error), and asserts the retry count or absence thereof as appropriate.

### Criterion 11
**Status:** FAIL
**Evidence:** spawn.test.ts:361 'dispose_all' test spawns only 2 children and does it via runtime.spawn_cli rather than concurrent engine.generate calls. The criterion requires N≥5 concurrent engine.generate calls, abort a subset mid-stream, then dispose; none of these three properties (≥5, concurrent-at-engine-level, abort-subset) hold.
**Required state:** A named test exists that creates an engine via create_engine, fires N≥5 concurrent engine.generate calls against hanging mock children, aborts a non-empty proper subset of them mid-stream, calls engine.dispose(), and asserts every child has exited (killed===true or exitCode!==null) and the live registry is empty.

### Criterion 14
**Status:** FAIL
**Evidence:** See criterion 4. No test invokes the adapter end-to-end with auth_mode:'oauth' + provider config api_key + call_opts.env.ANTHROPIC_API_KEY and inspects the mock's recorded env. mock_claude.mjs supports MOCK_CLAUDE_RECORD and the infrastructure exists (spawn.test.ts uses it for a different assertion), but the auth-scrub test uses build_env directly.
**Required state:** Same as criterion 4: one named test exercises the full adapter-driven spawn with oauth + config.api_key + call_opts.env.ANTHROPIC_API_KEY and asserts the subprocess's observed env does not contain ANTHROPIC_API_KEY.
