# Phase 1: Claude CLI Provider Adapter

## Goal

Deliver the complete `claude_cli` subprocess provider so that a caller can write `engine.generate({ model: 'cli-sonnet', prompt })` against a `create_engine` whose `providers` map contains a `claude_cli` entry, and the engine spawns the local `claude` binary, streams its `stream-json` output back as `StreamChunk`s, and returns a `GenerateResult` whose shape matches the HTTPS Anthropic adapter except for `model_resolved.provider === 'claude_cli'`, `provider_reported.claude_cli` carrying `{ session_id, duration_ms }`, and trajectory `cost` events carrying `source: 'provider_reported'` (with cost sourced from the CLI's `total_cost_usd`).

This phase owns every source file under `packages/engine/src/providers/claude_cli/` (types, constants, spawn, argv, stream_parse, stream_result, auth, sandbox, cost, index), the small wiring edits elsewhere in the engine that register the adapter and surface its public types (`packages/engine/src/types.ts`, `packages/engine/src/aliases.ts`, `packages/engine/src/providers/registry.ts`, `packages/engine/src/index.ts`), and the new architectural ast-grep guardrails that mechanically enforce the boundary invariants from spec §13 and constraints §7.16–§7.25 from the moment the directory exists.

Subprocess lifecycle (detached process groups, per-adapter live-child registry, SIGTERM→SIGKILL escalation, startup/stall timers, synchronous Node-exit reap), `auth_mode` env scrubbing, tool-bridge modes (`'allowlist_only'` default, `'forbid'`), schema repair via `--resume`, sandbox wrapping (bwrap, greywall), per-turn output-token-weighted cost allocation, and the universal `Engine.dispose()` contract all land here. The engine's `generate.ts` already dispatches on `adapter.kind === 'subprocess'`; this phase verifies the dispatch and adds no provider-specific branches there.

## Context

The engine layer (`packages/engine/`) ships HTTPS-backed providers (Anthropic, OpenAI, Google, Ollama, LM Studio, OpenRouter) via Vercel AI SDK v6. The discriminated `ProviderAdapter` union in `packages/engine/src/providers/types.ts` already defines a `SubprocessProviderAdapter` branch with `{ kind: 'subprocess', name, generate, dispose, supports }`, and `packages/engine/src/generate.ts` already branches on `adapter.kind === 'subprocess'` to delegate to `adapter.generate(opts, resolved)`. The `Engine.dispose()` aggregator in `packages/engine/src/create_engine.ts` already iterates adapters whose `kind === 'subprocess'` and awaits each `dispose()`. This phase adds the first concrete `SubprocessProviderAdapter` implementation; no changes to the dispatch or aggregator are required beyond imports.

The `errors.ts` module in `packages/engine/src/` already exports `claude_cli_error`, `provider_auth_error`, `provider_capability_error`, `engine_config_error`, `engine_disposed_error`, `aborted_error`, and `schema_validation_error` (per constraints §6 and the recent commit history). No new typed errors are introduced.

Project-wide constraints that apply here without restatement: no `class`/`extends`/`this` outside `packages/engine/src/errors.ts`; no `process.env` or `process.cwd()` reads in any `packages/engine/src/**` file; named exports only; snake_case for values, PascalCase for type aliases; ESM-only; Zod 4's `z.toJSONSchema()` (no `zod-to-json-schema` dependency); `node:`-prefixed Node built-ins. The full check pipeline (`pnpm check`) must exit zero at the end of this phase.

Prior art for pattern reference (do not copy verbatim): `../../../../../../../ridgeline/code/ridgeline/src/engine/claude/{claude.exec,stream.parse,stream.types,stream.result,sandbox,sandbox.bwrap,sandbox.greywall}.ts`. The shapes there are correct; the code is not. Reimplement against this spec, not against the prior art.

## Acceptance Criteria

### Source files exist with the documented shapes

1. `packages/engine/src/providers/claude_cli/types.ts` exports `ClaudeCliProviderConfig`, `ClaudeCliCallOptions`, `AgentDef`, `SandboxProviderConfig`, `AuthMode`, `ToolBridgeMode`, and `ClaudeCliProviderReported` with the exact field shapes and optionality documented in spec §5.1, §5.3, and §5.4.
2. `packages/engine/src/providers/claude_cli/constants.ts` exports `Object.freeze`-d `CLI_AUTH_ERROR_PATTERNS` (case-insensitive matchers covering `authentication`, `unauthorized`, `forbidden`, `oauth token has expired`, `invalid_api_key`), `CACHE_READ_MULTIPLIER` (`0.1`), `CACHE_WRITE_MULTIPLIER` (`1.25`), `DEFAULT_STARTUP_TIMEOUT_MS` (`120_000`), `DEFAULT_STALL_TIMEOUT_MS` (`300_000`), `SIGKILL_ESCALATION_MS` (`2000`), and `CLI_BINARY_DEFAULT` (`'claude'`).
3. `packages/engine/src/providers/claude_cli/argv.ts` exports a pure argv-building function returning a readonly string array that emits, in order: `-p`, `--output-format stream-json`, `--model <model_id>`, `--verbose`, `--setting-sources <comma-joined>`, then `--allowedTools <name>` per merged tool, `--resume <session_id>` when present, `--agents <json>` when present, `--plugin-dir <path>` per dir, `--json-schema <compiled>` when schema present, `--append-system-prompt <merged>` when system or `append_system_prompt` is non-empty, then `extra_args` verbatim. Every argv element is a separate array entry; no string interpolation of option values (e.g. no `` `--model=${id}` ``).
4. `packages/engine/src/providers/claude_cli/auth.ts` exports `build_env(config, caller_env, auth_mode)` that never reads `process.env` and returns a fresh `Record<string, string>`: under `'oauth'` `ANTHROPIC_API_KEY` is stripped from both config-derived and caller-supplied sources; under `'api_key'` `ANTHROPIC_API_KEY` is set from `config.api_key`; under `'auto'` `ANTHROPIC_API_KEY` is set only when `config.api_key` is present. Also exports a stderr matcher that tests against `CLI_AUTH_ERROR_PATTERNS` case-insensitively and a synchronous validator that throws `engine_config_error('api_key is required for auth_mode: api_key', 'claude_cli')` when `auth_mode === 'api_key'` and `api_key` is empty/missing.
5. `packages/engine/src/providers/claude_cli/sandbox.ts` exports a pure wrapper that, given a `SandboxProviderConfig` and a binary path, returns the spawn command and argv-prefix pair for `bwrap` and `greywall`, honoring `network_allowlist` and `additional_write_paths`. Patterns are ported from ridgeline prior art, not copied verbatim.
6. `packages/engine/src/providers/claude_cli/cost.ts` exports a decomposition function that, given `total_cost_usd` and a `UsageTotals`, returns a `CostBreakdown` whose `input_usd + output_usd + cached_input_usd + cache_write_usd` equals `total_usd` within `1e-9`, using `CACHE_READ_MULTIPLIER` and `CACHE_WRITE_MULTIPLIER` from constants, with `is_estimate: true`. Also exports a per-turn allocation helper that splits `total_cost_usd` across turns proportional to each turn's `output_tokens` such that `sum(steps[i].cost.total_usd) === total_usd` exactly.
7. `packages/engine/src/providers/claude_cli/stream_parse.ts` exports a line-buffered JSON-lines parser that accumulates partial lines until `\n`, records `{ kind: 'cli_parse_error', line }` to trajectory for malformed JSON and continues, records `{ kind: 'cli_unknown_event', raw }` for unknown `type` values and continues, and maps each recognized event per the table in spec §7.2 into `StreamChunk`s. `step_index` increments whenever an `assistant` event follows a `user`/`tool_result` event; the first `assistant` event is step `0`. `tool_use` content emits `tool_call_start` followed atomically by `tool_call_end` with full input (no `tool_call_input_delta`).
8. `packages/engine/src/providers/claude_cli/stream_result.ts` converts the terminal CLI `result` event into a `GenerateResult` whose `usage` maps `input_tokens` → `input_tokens`, `output_tokens` → `output_tokens`, `cache_read_input_tokens` → `cached_input_tokens`, `cache_creation_input_tokens` → `cache_write_tokens` (`reasoning_tokens` undefined); cost is populated via the §10 decomposition; `provider_reported.claude_cli = { session_id, duration_ms }`; `cost.is_estimate` is true.
9. `packages/engine/src/providers/claude_cli/spawn.ts` spawns via `node:child_process` with `{ detached: true, stdio: ['pipe', 'pipe', 'pipe'], env, cwd, shell: false }` always explicit and an `env` built by `build_env` (never inheriting `process.env`); holds a per-adapter-factory closure-captured `Set<ChildProcess>`; every spawn inserts the child and every `'close'` event removes it; the SIGTERM→SIGKILL escalator fires `process.kill(-pid, 'SIGTERM')` then `process.kill(-pid, 'SIGKILL')` after `SIGKILL_ESCALATION_MS` if the child has not exited (the SIGKILL throw is caught and ignored); a `process.on('exit')` handler synchronously issues `process.kill(-pid, 'SIGKILL')` for every live-set member with no `await`, wrapped in try/catch; startup timer (default 120s) is armed at spawn and cleared on first stdout byte; stall timer (default 300s) resets on every stdout chunk; both reject with `claude_cli_error('startup_timeout', ...)` or `claude_cli_error('stall_timeout', ...)` and are independent from `generate_options.abort`.
10. `packages/engine/src/providers/claude_cli/index.ts` exports a factory returning exactly `{ kind: 'subprocess', name: 'claude_cli', generate, dispose, supports }` — no `build_model`, `translate_effort`, or `normalize_usage` members. The factory captures the live-child registry by closure (no module-level mutable state); two factory invocations produce two independent registries.

### Wiring edits

11. `packages/engine/src/types.ts` extends `ProviderConfigMap` so `claude_cli?: ClaudeCliProviderConfig` is permitted; `packages/engine/src/index.ts` re-exports `ClaudeCliProviderConfig`, `ClaudeCliCallOptions`, `AgentDef`, `SandboxProviderConfig`, `AuthMode`, `ToolBridgeMode`, and `ClaudeCliProviderReported`.
12. `packages/engine/src/aliases.ts` `DEFAULT_ALIASES` includes `cli-opus` → `{ provider: 'claude_cli', model_id: 'claude-opus-4-7' }`, `cli-sonnet` → `{ provider: 'claude_cli', model_id: 'claude-sonnet-4-6' }`, `cli-haiku` → `{ provider: 'claude_cli', model_id: 'claude-haiku-4-5' }`; `KNOWN_PROVIDERS` contains `'claude_cli'`.
13. `packages/engine/src/providers/registry.ts` exports a `create_claude_cli_adapter` factory wired into the registry such that a `create_engine({ providers: { claude_cli: {...} } })` constructs and registers the adapter.
14. `packages/engine/src/generate.ts` requires no source changes beyond imports/types; the existing `adapter.kind === 'subprocess'` dispatch continues to delegate to `adapter.generate(opts, resolved)` without a provider-specific branch.

### Behavior on a working call

15. Against a shell-script fixture (or `node:child_process.spawn` mock) that emits a single `result` event, `engine.generate({ model: 'cli-sonnet', prompt: 'hi' })` resolves with a `GenerateResult` where `model_resolved.provider === 'claude_cli'`, `finish_reason === 'stop'`, `cost.total_usd` equals the fixture's `total_cost_usd`, `cost.is_estimate === true`, `usage.cached_input_tokens` is mapped from `cache_read_input_tokens`, `usage.cache_write_tokens` is mapped from `cache_creation_input_tokens`, and `provider_reported.claude_cli.session_id` plus `duration_ms` are present.
16. Trajectory `cost` events carry `source: 'provider_reported'`; no `pricing_missing` event is ever emitted for `claude_cli`; per-turn allocation across multi-turn fixtures sums exactly to `total_cost_usd`.
17. `auth_mode: 'oauth'` strips `ANTHROPIC_API_KEY` from the spawned env even when both `ClaudeCliProviderConfig.api_key` and `ClaudeCliCallOptions.env.ANTHROPIC_API_KEY` are supplied.
18. `auth_mode: 'api_key'` without `api_key` in config throws `engine_config_error` synchronously inside `create_engine`.
19. Stderr matching any pattern in `CLI_AUTH_ERROR_PATTERNS` produces `provider_auth_error({ provider: 'claude_cli', refresh_command: 'claude login' })`.
20. `Tool[]` with `execute` closures under the default `tool_bridge: 'allowlist_only'` forwards each `tool.name` via `--allowedTools`, never invokes `execute`, and records exactly one `{ kind: 'cli_tool_bridge_allowlist_only', dropped: [...] }` trajectory event per call. Under `tool_bridge: 'forbid'`, the same input throws `provider_capability_error('claude_cli', 'tool_execute', ...)` before spawn.
21. Multi-user `Message[]` history (≥ 2 user messages) throws `provider_capability_error('claude_cli', 'multi_turn_history', 'use provider_options.claude_cli.session_id instead')` before spawn.
22. Zod schemas compile via `z.toJSONSchema(opts.schema)` and forward as `--json-schema`; on invalid `result.result` parse, exactly one repair call is issued with `--resume <session_id>` and a repair prompt; second failure throws `schema_validation_error`.
23. `provider_options.claude_cli.agents` forwards as `--agents <json>`; `plugin_dirs` forwards as N `--plugin-dir <path>` pairs; `extra_args` appends verbatim; `allowed_tools` grammar strings (e.g. `'Bash(git:*)'`, `'Write(/tmp/**)'`) pass through unparsed.
24. Sandbox config `{ kind: 'bwrap', ... }` and `{ kind: 'greywall', ... }` produces a spawn where the sandbox binary is `spawn_cmd` and `claude` appears in argv after the sandbox's own arguments; missing sandbox binary surfaces as `claude_cli_error('sandbox_unavailable', ...)` on first call.
25. Caller-supplied options the provider cannot honor (`max_steps`, `tool_error_policy`, `on_tool_approval`) each emit exactly one `{ kind: 'option_ignored', option, provider: 'claude_cli' }` trajectory event per option per call; `finish_reason: 'max_steps'` cannot occur for this provider.
26. Aborting mid-stream causes SIGTERM to the child's process group; SIGKILL fires after `SIGKILL_ESCALATION_MS` if the child has not exited; `generate` rejects with `aborted_error({ step_index, tool_call_in_flight? })`. No partial `GenerateResult` is returned even when `on_chunk` has already observed text.
27. `engine.dispose()` signals every live child, awaits each `close`, and resolves; in-flight `generate` promises reject with `aborted_error({ reason: 'engine_disposed' })`; a subsequent `engine.generate(...)` throws `engine_disposed_error` synchronously (caught via `try/catch`, not `await`); two engines disposing independently do not affect each other's children.
28. Binary missing on first call throws `claude_cli_error('binary_not_found', ...)` (probe is lazy, not at construction unless `skip_probe === false` is explicitly opt-in for early-fail).

### Architectural guardrails enforced mechanically

29. `rules/no-child-process-outside-claude-cli.yml` exists and fails `pnpm check` when `'node:child_process'` or `'child_process'` is imported from any file outside `packages/engine/src/providers/claude_cli/**`.
30. `rules/no-provider-sdk-in-claude-cli.yml` exists and fails `pnpm check` when `'ai'`, `'@ai-sdk/*'`, `'ai-sdk-ollama'`, or `'@openrouter/ai-sdk-provider'` is imported from any file under `packages/engine/src/providers/claude_cli/**`. Importing `'zod'` (including `z.toJSONSchema`) is permitted.
31. Either `rules/no-process-env-in-core.yml` is extended to also match `process.cwd` (regex extended to `^process\.(env|cwd)$`), or a sibling `rules/no-process-cwd-in-engine.yml` exists scoped to `packages/engine/src/**`. `pnpm check` fails on any `process.env` or `process.cwd()` read inside `packages/engine/src/**`.
32. `scripts/check-deps.mjs` (extended if necessary) verifies `@robmclarty/engine`'s production dependencies are exactly `'ai'` and `'zod'`, and that all provider SDK packages appear only under `peerDependencies` with `peerDependenciesMeta.<pkg>.optional: true`.

### Final gate

33. `pnpm check` exits zero with all source, wiring, and guardrails in place. All existing tests continue to pass; `tsc --noEmit` reports no new errors; oxlint passes; ast-grep passes (existing rules plus the new ones); cspell and markdownlint pass; `scripts/check-deps.mjs` passes.
34. No `class`, `extends`, or `this` appears anywhere in `packages/engine/src/providers/claude_cli/**`; the only files containing `class` in the repo remain `packages/core/src/errors.ts` and `packages/engine/src/errors.ts`.
