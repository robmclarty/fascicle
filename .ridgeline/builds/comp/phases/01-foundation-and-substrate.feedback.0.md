# Reviewer Feedback: Phase 01-foundation-and-substrate

## Failed Criteria

### Criterion 16
**Status:** FAIL
**Evidence:** All 35 tests pass (verified). However, the SIGINT cleanup test is in-process (uses process.emit('SIGINT') from within vitest) rather than the child-process harness explicitly required by criterion 16 and constraints.md §9. packages/core/test/cleanup/ does not exist; no child script is spawned; no slow fetch is exercised; no marker file is used; no process exit code is checked. The test verifies part of the contract (cleanup runs, aborted_error is the abort reason) but not the operational reality of an OS-delivered SIGINT. Remaining sub-tests (atomic step, anon id, describe, streaming order, streaming equivalence, streaming drop, cleanup LIFO, cleanup error, aborted reason typing, idempotent signal handlers, flow_schema example validation) all pass.
**Required state:** A child-process harness must live under packages/core/test/cleanup/. It spawns a subprocess that runs a flow with a long-running step (slow fetch or equivalent I/O), installs a cleanup handler that writes a marker file, and awaits SIGINT. The parent test sends SIGINT to the child, waits for the child to exit, and asserts: (a) the marker file exists, (b) the child exited non-zero, and (c) the in-flight I/O saw an aborted_error as its AbortSignal.reason (or equivalent observable evidence, e.g. a second marker file capturing the abort reason). Whether this is implemented by adding tsx to devDependencies, building to .check/dist first, or writing the child script in plain .mjs that imports from the package entry is up to the builder — but the test must exercise a real SIGINT across a process boundary, not process.emit inside vitest.
